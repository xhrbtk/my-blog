##7谈谈你对mvc mvp mvvm的理解
```
web1.0
web2.0 --- ajax出现 前后端分离
mvc mvp mvvm 
mvc: view controller model ;model 负责保存应用数据，与后端数据进行同步；controller 负责业务逻辑，根据用户行为对model数据进行更改；view负责视图展示，将model中数据可视化出来
mvp: p指的是presenter 可以理解为一个中间人 它负责view和model之间的数据流动，防止view和model之间的直接交流

这三者都是框架模式，他们设计的目标都是为了解决model和view的耦合问题。mvc模式出现较早主要应用在后端，如spring mvc asp.net mvc 等，在前端领域的早期也有应用跟，如backbone.js 。它的优点是分层清晰，缺点是数据流混乱，灵活性带来的维护性问题。
mvp模式在是mvc的进化形式，presenter作为中间层负责mv通信，解决了两者耦合问题，但是p层过于臃肿会导致维护问题
mvvm模式在前端领域有广泛应用，它不仅解决mv耦合问题，还同事解决了维护两者映射关系的大量繁杂代码和dom操作代码，在提高开发效率，可读性同事还保持了优越性能表现
```





## 8vue 优化
```
1.使用v-show复用dom
2.v-for 遍历避免同事使用v-if
3.长列表性能优化- 如果列表是纯粹的数据展示，不会有任何改变，就不需要做响应化Object.freeze。如果是大数据长列表，可采用虚拟滚动，只渲染少部分内容
4.图片懒加载
5.第三方插件按需引入
6.无状态组件标记为函数式组件
7.自组件分割
8.ssr
```
